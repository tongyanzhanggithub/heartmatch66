# HeartMatch 💕 相亲活动管理后台

一套面向线下相亲活动运营者（红娘）的私有化管理系统：嘉宾资料、报名审核、活动排期、现场签到、复盘财务、AI 匹配、八字合盘，全流程集中管理，替代散落的 Excel 和微信记录。

> 内部自用系统 · 云端部署 · 手机端友好

## ✨ 功能总览

### 嘉宾侧（手机报名页 `/apply/`）
- 6 步引导式报名表单，适配手机触屏
- 收集 60+ 字段：基本信息、职业背景、性格标签（MBTI/个性/运动/生活方式/价值观，支持自定义标签）、快问快答、结构化择偶条件、出生日期时辰（八字用）
- 单身承诺 + 免责协议电子签署，脱敏展示/肖像宣传分项授权
- IP 限流 + 同微信号 24h 去重防刷

### 运营侧（管理后台 `/`）
| 模块 | 能力 |
|------|------|
| **嘉宾审核** | 待审/待补/拒绝三队列，核验勾选（实名/证件/单身承诺），审核留痕，闭环流转 |
| **嘉宾管理** | 男女分栏 + 黑名单库，全字段搜索，红娘标签，Excel/CSV 导入导出（66 列） |
| **标签分群** | 自定义标签（小美/顶美…）与圈层分群，批量打标 |
| **活动管理** | 列表/日历双视图，名额票价管理，男女比例实时显示 |
| **智能荐人** | 按与已报名异性的双向匹配分排序，圈层加权，批量加入报名 |
| **现场签到** | 全屏大按钮签到模式，实时到场率，手机操作友好 |
| **复盘财务** | 票房按已付费×票价自动算，净利/到场率自动核算 |
| **AI 双向匹配** | 双向评分（调和平均）+ 共性维度 + 硬性条件过滤 + 贝叶斯覆盖度校准 |
| **八字合盘** | 真实排盘（lunar-javascript），生肖/五行/日柱/星座四维解读，缘分关键词，黄历约会吉日，精美分享卡 |
| **脱敏资料卡** | 一键生成对外预热卡图片，未授权自动拦截（合规内置） |
| **仪表盘** | 月度净利/到场率趋势图，待办提醒 |

## 🏗️ 技术栈

```
├── server/   Node.js + Express 5 + better-sqlite3 + JWT     # 后端 API
├── client/   React 19 + Vite + Tailwind + Recharts          # 管理后台
└── mobile/   React 19 + Vite + Tailwind                     # 手机报名页
```

- 数据库：SQLite 单文件（`server/data.db`），每日自动备份保留 30 份
- 认证：JWT（7 天有效期），bcrypt 密码哈希，登录防爆破（5 次锁 15 分钟）
- 部署：Docker Compose 三段式构建（已适配国内镜像源）

## 🚀 快速开始

### 本地开发

```bash
# 后端（端口 3001）
cd server && npm install
cp .env.example .env        # 修改 JWT_SECRET 和管理员密码
npm run dev

# 管理后台（端口 5173）
cd client && npm install && npm run dev

# 手机报名页（端口 5174，路径 /apply/）
cd mobile && npm install && npm run dev
```

### 云端部署（Docker）

```bash
cp .env.production.example .env   # 填写生产密钥（务必修改！）
docker compose up -d --build
```

部署后：管理后台 `http://服务器:3001/`，报名页 `http://服务器:3001/apply/`。
完整步骤（含阿里云国内环境踩坑、Nginx/HTTPS、备份恢复）见 [部署说明.md](部署说明.md)。

```bash

node scripts/add-admin.js 用户名 密码    # 新增/重置管理账号
```

## 📋 环境变量（server/.env）

| 变量 | 说明 |
|------|------|
| `PORT` | 服务端口，默认 3001 |
| `JWT_SECRET` | 登录令牌密钥（32 位以上随机串，**部署必改**） |
| `ADMIN_USERNAME` / `ADMIN_PASSWORD` | 初始管理员（首次建库时写入） |

## 🔒 隐私与合规设计

- 敏感证件「只核验不留存」：仅记录核验结果标记，不存证件原件
- 对外展示强制脱敏：生成嘉宾卡需本人勾选授权，未授权拦截
- 真实姓名/联系方式仅管理员登录可见，不出现在任何对外输出
- `data.db`、备份、`.env` 均已在 `.gitignore`，不会进入版本库
