# 相亲活动管理后台 — 部署与使用说明

## 系统组成

| 模块 | 目录 | 说明 |
|------|------|------|
| 后端 API | `server/` | Node.js + Express + SQLite，含定时备份 |
| 管理后台 | `client/` | React 网页，运营者使用（电脑端） |
| 手机报名页 | `mobile/` | 嘉宾自助填资料，部署后路径为 `/apply/` |

数据库为单文件 `server/data.db`，每天自动备份到 `server/backups/`（保留最近 30 份）。

---

## 本地开发启动

```bash
# 终端1 - 后端（3001端口）
cd server && npm install && npm run dev

# 终端2 - 管理后台（5173端口）
cd client && npm install && npm run dev

# 终端3 - 手机报名页（5174端口，可选）
cd mobile && npm install && npm run dev
```

- 管理后台：http://localhost:5173
- 手机报名页：http://localhost:5174/apply/
- 默认账号：`admin` / `Admin@2024!`（在 `server/.env` 中修改）

## 云端部署（Docker，推荐）

### 前提
- 云服务器（1核2G即够）已安装 Docker + Docker Compose
- 开放 3001 端口（或用 Nginx 反代 80/443）

### 步骤

```bash
# 1. 上传整个项目目录（不含 node_modules）
scp -r . root@服务器IP:/opt/xiangqin

# 2. 登录服务器配置环境变量
ssh root@服务器IP
cd /opt/xiangqin
cp .env.production .env
nano .env        # 必须修改 JWT_SECRET 和 ADMIN_PASSWORD！

# 3. 构建并启动（会自动构建管理后台 + 手机端）
docker compose up -d --build

# 4. 查看日志确认启动成功
docker compose logs -f
```

### 部署后的访问地址
| 用途 | 地址 |
|------|------|
| 管理后台（自己用） | `http://服务器IP:3001/` |
| 嘉宾报名页（发给嘉宾） | `http://服务器IP:3001/apply/` |

## 云端部署（无 Docker，PM2）

```bash
# 安装 Node.js 20+
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash - && sudo apt install -y nodejs

# 构建两个前端
cd client && npm ci && npm run build && cd ..
cd mobile && npm ci && npm run build && cd ..

# 后端
cd server && npm ci --omit=dev
cp .env.example .env && nano .env   # 改密钥和密码

# PM2 守护
npm install -g pm2
NODE_ENV=production pm2 start index.js --name xiangqin
pm2 save && pm2 startup
```

## Nginx + 域名 + HTTPS（推荐，微信里打开更可信）

```nginx
server {
    listen 80;
    server_name 你的域名.com;
    location / {
        proxy_pass http://localhost:3001;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

```bash
sudo certbot --nginx -d 你的域名.com   # 自动配置 HTTPS
```

之后报名页就是 `https://你的域名.com/apply/`，可生成二维码发群里。

## 数据管理

### 自动备份
- 服务启动 1 分钟后备份一次，之后每 24 小时一次
- 备份位置：`server/backups/data_日期时间.db`，保留最近 30 份
- 手动立即备份：`cd server && npm run backup`

### 恢复备份
```bash
cd server
# 停服务后，用备份覆盖主库
cp backups/data_2026-06-12-10-00-00.db data.db
# 重启服务
```

### 上线前清空测试数据
```bash
cd server && npm run reset-data
# 会先自动备份一次，然后清空嘉宾/活动/报名/复盘，管理员账号保留
```

### 导入老数据
管理后台 → 嘉宾管理 → 「模板」下载 Excel 模板 → 按模板整理老数据 → 「导入」。
导出的文件可修改后再导入（同微信号/同昵称自动去重）。

## 环境变量（server/.env）

| 变量 | 说明 |
|------|------|
| `PORT` | 后端端口，默认 3001 |
| `JWT_SECRET` | 登录令牌密钥，**部署前必须改成32位以上随机字符串** |
| `ADMIN_USERNAME` | 管理员用户名 |
| `ADMIN_PASSWORD` | 管理员密码，**部署前必须改强密码** |

> 注意：管理员账号在数据库首次初始化时创建。改 `.env` 中的密码后，如果库里已有旧账号，需删除 `data.db` 重新初始化，或保持原密码登录。
